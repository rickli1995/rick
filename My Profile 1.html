Tank War in Unity - 90th Edition Classic Game Reenactment.

As a classic game in the era of red and white machines, the 90 tank is definitely one of the representative works of the year. 

1. Scene construction

First, import the prepared scene material into the project, set the tag of the brick wall to Wall, the grass tag to Wall1, the water wall to Wall2, the wall Wall3, and add the Box Collider2D component to each object. An empty node for mounting enemies and players and bullets.

2. Data configuration table

As an expandable and configurable tank battle, of course, you can't do without the configuration table settings. You can add all the character attributes or prop properties you want to have in the configuration table.

Player table analysis design:

1. A unique ID is needed to distinguish the data in the table to facilitate indexing;

2. A level is required to bind the display and bullet type;

3. Need a picture name to load the image;

4. Need bullet type, used to display different bullet effects, and can penetrate the iron wall and can not;

Enemy table analysis design:

1. A unique ID is needed to distinguish the data in the table to facilitate indexing;

2. A level is required to bind the display and bullet type;

3. Need a picture name to load the picture;

4. Need bullet type, used to display different bullet effects, and can penetrate the iron wall and can not;

Prop analysis:

1. Add ID to facilitate indexing;

2. Add a picture name to display the shape of the item;

After making the form, I need to put the form into Unity to read. 3.GameMode

When the data is acquired, the scene loading starts, you need to load the player character, enemy characters, props information, etc., and convert their corresponding table information into dynamic information that can be called at any time.

4. Player

Simply build a player model and design player behavior: The player can move in four directions: up, down, left, and right; Each movement distance is 1/2 of the width of a wall in the scene; Players cannot cross obstacles; Players can fire bullets; Players can get items; The player can change the shape and dictionary effect after getting the item upgrade; The player can die;

5. Enemy

Set up the enemy model and set the enemy Ai: The enemy can move in four directions: up, down, left, and right; Each movement distance is 1/2 of the wall in the scene; The enemy cannot cross the obstacle; if there is no obstacle, randomly choose whether to continue to advance in the current direction, and if the obstacle is encountered, randomly select the moving direction; The enemy can fire bullets at a frequency of 1.5s once; The enemy can die;
